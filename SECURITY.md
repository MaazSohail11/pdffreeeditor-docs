# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in PDF Free Editor or its documentation, we ask that you report it responsibly.

### How to report

- **Email:** [contact@pdffreeeditor.com](mailto:contact@pdffreeeditor.com)
- **Contact page:** [https://pdffreeeditor.com/contact/](https://pdffreeeditor.com/contact/)

Please include:

1. A description of the vulnerability and its potential impact.
2. Steps to reproduce the issue (if applicable).
3. Any relevant screenshots or logs (redact sensitive information).

### What to expect

- **Acknowledgement** — We will confirm receipt of your report within **48 hours**.
- **Investigation** — We will investigate the issue and keep you informed of progress.
- **Resolution** — We aim to resolve confirmed vulnerabilities promptly and will notify you when a fix is deployed.

### Safe harbor

We consider security research conducted in good faith to be authorized. We will not pursue legal action against individuals who:

- Make a good-faith effort to avoid privacy violations, data destruction, and service disruption.
- Report vulnerabilities to us before disclosing them publicly.
- Give us a reasonable amount of time to address the issue before public disclosure.

### Scope

This policy covers the web application at [pdffreeeditor.com](https://pdffreeeditor.com) and the documentation in this repository. If you are unsure whether an issue is in scope, please report it anyway — we appreciate the effort.

Thank you for helping keep PDF Free Editor secure. 🔒
