# Contributing to PDF Free Editor Docs

Thank you for your interest in improving this documentation! Contributions are welcome.

## How to contribute

1. **Fork** this repository.
2. **Create a branch** for your change (`git checkout -b docs/improve-compress-guide`).
3. **Make your edits** — fix typos, clarify steps, add examples, or improve diagrams.
4. **Commit** with a clear message (e.g., `docs: clarify rasterize vs compact in compress-pdf`).
5. **Open a pull request** describing what you changed and why.

## Guidelines

- **Keep it user-focused.** Write for someone who may never have used a PDF tool before.
- **Avoid marketing fluff.** Be factual and helpful; let the tool speak for itself.
- **Use natural anchor text** when adding links (e.g., "Compress PDF online" not raw URLs).
- **One main link + 1–2 supporting links** per workflow doc. Don't overload pages with links.
- **Match existing tone.** These docs are practical and conversational, not academic.

## Reporting issues

If you find incorrect information, broken links, or confusing instructions, please [open an issue](https://github.com/MaazSohail11/pdffreeeditor-docs/issues).

## Code of conduct

Please follow our [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions.
